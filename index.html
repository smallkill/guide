把 Codex 接上 Telegram,
打造你自己的 AI 助理

用 OpenAI 的 Codex CLI 加上一個 Telegram bot,就能從手機隨時下指令、看進度。這篇用最少廢話帶你走完:原生外掛系統、現成的橋接工具、真實安裝步驟,還有最容易讓人吃虧的計費與安全。

閱讀約 6 分鐘 難度:會用終端機即可 適用:Codex CLI

TL;DR 先講結論

  • Codex 有原生外掛系統(2026 年 3 月推出),CLI 裡用 /plugins 安裝,底層走 MCP,官方加社群已有上百個。
  • 但目前沒有官方的 Telegram 外掛。要接 Telegram,得用社群做的橋接程式(本文介紹三個),或在 /plugins 裡找社群版。
  • 架構很單純:Telegram bot 收到訊息,橋接程式餵給 Codex,把回覆再傳回去。最省事的選擇是 TeleCodex。
  • 最大的坑是計費。一定要跑在 ChatGPT 訂閱方案,而不是「每則訊息呼叫一次 API key」,否則帳單會失控。
01

它是怎麼運作的

Codex CLI 本身是一個跑在你電腦終端機裡的 AI agent(類似 Claude Code)。它不知道 Telegram 的存在。中間需要一個「橋接程式」當翻譯。

流程拆開來看:你在手機 Telegram 打一句話,訊息經由你向 BotFather 申請的 bot 進來,橋接程式收到後把它丟給本機的 Codex 執行,Codex 跑完再把結果透過 Telegram Bot API 傳回你手機。橋接程式不會去改動 Codex 本身,它只是一層薄薄的轉接。

02

三個現成的橋接工具

不用自己從零寫。社群已經做好幾個開源橋接,挑一個跟著它的 README 裝就好。以下是目前較成熟的三個。

TeleCodex

最推薦上手

專為 Codex CLI 做的 Telegram 橋接,文件最完整。支援每個聊天室各自獨立 session、即時串流回應與工具輸出、語音和圖片輸入,還能用 /handback 把對話交回終端機繼續。下一節有完整安裝步驟。

github.com/benedict2310/telecodex ↗

OpenAB

輕量

Open Agent Bridge。用 Python 包一層,實際上是呼叫 codex exec 和 codex exec resume --last 來續接對話。Telegram 和 Discord 都支援,一個設定檔就跑得起來。

github.com/xx025/openab ↗

Untether

多 agent 通吃

同時支援 Codex CLI、Claude Code、Gemini CLI 等多種 agent。可以用文字或語音派任務、即時看進度。如果你想之後換別的 agent,選這個最有彈性。

littlebearapps.com/builds/untether ↗

03

動手做:用 TeleCodex 接起來

以下步驟對應 TeleCodex 的官方 README。實際裝的時候請以該專案最新說明為準,版本更新可能調整。

準備環境

需要 Node.js 22 以上,以及已經安裝並登入過的 Codex CLI。想用語音轉文字的話,額外裝 ffmpeg。

向 BotFather 申請一個 bot

在 Telegram 搜尋 @BotFather,送出 /newbot,照指示取名,它會給你一組 bot token。先複製起來。

下載並安裝

終端機bash
# 取得專案後,在專案資料夾內
npm install
cp .env.example .env

填寫 .env 設定

打開 .env,至少要填這兩個:

.env環境變數
TELEGRAM_BOT_TOKEN=# 上一步 BotFather 給的 token
TELEGRAM_ALLOWED_USER_IDS=# 你的 Telegram user id(逗號分隔)

TELEGRAM_ALLOWED_USER_IDS 是白名單,只有名單內的人能用你的 bot。這一欄一定要填,否則任何人找到你的 bot 都能操作你的電腦。

讓 Codex 登入

驗證方式二選一:在 Telegram 裡送 /login 走裝置授權流程(不必碰終端機),或在 .env 填 CODEX_API_KEY 走 API 金鑰。計費差很大,請先看第 5 節。

啟動

終端機bash
npm run dev

# 或用 Docker
docker compose up --build

跑起來後,在 Telegram 對你的 bot 說句話試試。它會把回應和工具使用即時串流回來。

i 可選的進階設定

TeleCodex 還能設定 CODEX_MODEL(預設模型)、CODEX_SANDBOX_MODE(沙盒權限:read-only / workspace-write / danger-full-access)、CODEX_APPROVAL_POLICY(動作核可策略)。新手建議從 唯讀或 workspace-write 開始,別一開始就開全權限。

04

順帶認識 Codex 的 /plugins 外掛系統

Telegram 目前靠橋接,但 Codex 其他能力可以直接用官方外掛擴充,這也是它「像 Claude 那樣」的地方。

Codex 在 2026 年 3 月推出了外掛市集,把技能、應用整合、MCP server 打包成可安裝的套件,App、CLI、IDE 擴充都能用。在 CLI 裡直接用指令安裝:

Codex CLI指令
/plugins

底層大多由 MCP(Model Context Protocol) 負責:每個 MCP server 是一個獨立程序,Codex 透過協定跟它溝通,支援自訂工具、具狀態的 session 和多步驟操作。官方已有的外掛包含 Slack、GitLab、Figma、Microsoft、Atlassian 等。

  • 想要「裝個官方外掛就有 Telegram」:目前還沒有,得用上面的第三方橋接。
  • 但社群外掛數量不少,可以在 /plugins 裡搜 telegram 看有沒有社群做好的版本。
05

上線前一定要知道的事

這節是整篇最重要的。前面裝錯頂多跑不起來,這裡踩雷會傷荷包或開後門。

! 計費:訂閱 vs 每則訊息呼叫 API

Codex 可以跑在 ChatGPT 訂閱方案(Plus / Pro 現在含 Codex),也可以用 API 金鑰按 token 計費。

如果你用 API 金鑰,又讓橋接程式「每收到一則訊息就開一次新呼叫」,在背景跑久了帳單會非常可觀。務必確認你是用訂閱登入,或維持單一常駐 session,不要狂開新呼叫。這是真的有人付出代價學到的教訓。

! 安全:你等於把電腦的鑰匙交出去

橋接程式讓 Telegram 可以操作你本機的 Codex,等於開了一條遠端通道。三個底線:

① 白名單必填:TELEGRAM_ALLOWED_USER_IDS 只放自己。
② 沙盒收緊:從 read-only 或 workspace-write 起步,別預設全權限。
③ 別把 token 貼到對話或公開倉庫:bot token 和 API key 都放在 .env,並確認 .gitignore 有排除它。

i 工具更新很快,以 README 為準

這些橋接都是社群維護的開源專案,設定欄位和指令可能隨版本變動。真正安裝時,以各專案 GitHub 上的最新 README 為準,本文是帶你建立全貌。

06

那我該選哪個

  • 只想最快接起來、單純用 Codex:選 TeleCodex,文件最齊、功能最完整。
  • 之後可能換別的 agent(Claude Code / Gemini):選 Untether,一套通吃。
  • 喜歡簡單、想自己改:選 OpenAB,Python 薄包裝,一個設定檔。

三條路的核心都一樣:一個 Telegram bot、一個橋接、一個本機 Codex。把計費和白名單顧好,你就有了一個從手機就能差遣的個人 AI 助理。
